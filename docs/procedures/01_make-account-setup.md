## Step 1: Slack デモワークスペースの作成

### 1-1. サインアップ

1. ブラウザで [https://slack.com/get-started](https://slack.com/get-started) を開く
2. 「メールアドレスで始める」をクリック
3. Google アカウントでサインインする（「Googleで続ける」を選択するとラク）
4. 「新しいワークスペースを作成する」を選択

### 1-2. ワークスペースの設定

1. **ワークスペース名**を入力する
   - 例: `portfolio-demo`（クライアントへの提案デモ用なのでシンプルな名前でよい）
2. 「何に使いますか？」は「フリーランス・コンサルタント」など適当に選ぶ
3. チームメンバーの招待は「後で」でスキップする
4. デフォルトのチャンネル（#general など）が作成された状態でダッシュボードに入る

### 1-3. 通知チャンネルの作成

1. 左サイドバーの「チャンネルを追加する」をクリック
2. 「チャンネルを作成する」を選択
3. 以下を設定する：
   - **チャンネル名**: `通知`（半角スペースなし）
   - **説明**: `フォーム回答の自動通知（Make連携）`
   - **公開設定**: パブリック
4. 「作成する」をクリック

> ポイント: チャンネル名は後で Make に設定するので正確に `通知` と入力する。

## Step 2: Google フォームの作成

### 2-1. フォームを新規作成

1. ブラウザで [https://forms.google.com](https://forms.google.com) を開く
2. 左上の「＋」（空白のフォーム）をクリックして新規作成
3. **タイトル**を入力する
   - 例: `お問い合わせフォーム（デモ）`
4. タイトル下の「フォームの説明」に `このフォームはポートフォリオデモ用です。` と入力する（任意）

### 2-2. 質問を3つ設定する

#### 質問1: 名前

1. デフォルトで「質問」が1つあるので、それをクリックして編集する
2. 質問文: `お名前`
3. 回答形式: `記述式（短文）`
4. 「必須」をオンにする

#### 質問2: メールアドレス

1. 「＋」ボタンで質問を追加する
2. 質問文: `メールアドレス`
3. 回答形式: `記述式（短文）`
4. 「必須」をオンにする

#### 質問3: お問い合わせ内容

1. 「＋」ボタンで質問を追加する
2. 質問文: `お問い合わせ内容`
3. 回答形式: `段落（長文）`
4. 「必須」をオンにする

### 2-3. フォームIDの取得

1. ブラウザのアドレスバーのURLを確認する
2. URL形式: `https://docs.google.com/forms/d/XXXXXXXXXXXXXXXXXXXX/edit`
3. `/d/` と `/edit` の間にある文字列が **フォームID**
   - 例: `1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms`
4. この値をメモしておく（Step 5 で記録）

## Step 3: Google スプレッドシートの作成

### 3-1. スプレッドシートを新規作成

1. ブラウザで [https://sheets.google.com](https://sheets.google.com) を開く
2. 左上の「＋」（空白のスプレッドシート）をクリックして新規作成
3. 左上のタイトル「無題のスプレッドシート」をクリックして名前を変更する
   - 名前: `フォーム回答一覧`

### 3-2. ヘッダー行の設定

1. A1セルから順に以下を入力する：

| A1 | B1 | C1 | D1 |
|----|----|----|-----|
| 名前 | メール | 内容 | 受信日時 |

2. ヘッダー行（1行目）を選択して背景色をグレーにすると見やすい（任意）

### 3-3. シート名の確認・変更

1. 画面下のシートタブを確認する（デフォルトは「シート1」）
2. シートタブを右クリック → 「名前を変更」を選択
3. **シート名**: `回答一覧` に変更する

> ポイント: Make でシート名を指定するので `回答一覧` と正確に入力する。

### 3-4. スプレッドシートIDの取得

1. ブラウザのアドレスバーのURLを確認する
2. URL形式: `https://docs.google.com/spreadsheets/d/XXXXXXXXXXXXXXXXXXXX/edit`
3. `/d/` と `/edit` の間にある文字列が **スプレッドシートID**
   - 例: `1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms`
4. この値をメモしておく（Step 5 で記録）

## Step 4: Make アカウントの作成

### 4-1. サインアップ

1. ブラウザで [https://www.make.com](https://www.make.com) を開く
2. 右上の「Get started free」または「Sign up free」をクリック
3. **「Continue with Google」を選択**（Google アカウントでサインアップが最もラク）
4. Google アカウントを選択して認証を進める

### 4-2. 初期設定

1. サインアップ後、「Tell us about yourself」の画面が出る場合は適当に選んで進める
2. ダッシュボードに「Scenarios」が表示されればセットアップ完了

### 4-3. 無料プランの制限について

Make の無料プランには以下の制限がある：

| 項目 | 制限 |
|------|------|
| 月間オペレーション数 | 1,000 回 |
| アクティブシナリオ数 | 2個 |
| データ転送 | 100 MB |
| 実行間隔 | 最短15分 |

> ポートフォリオのデモ・テスト目的であれば無料プランで十分。クライアントへの提案時は「本番運用では有料プランに移行するか、クライアント側のアカウントで設定します」と説明する。
