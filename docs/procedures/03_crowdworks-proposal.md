# CrowdWorks 提案手順書

フリーランス案件の発見〜提案送信〜フォローアップまでの詳細手順。
毎週月・水・金に1件ずつ、週3件ペースで実施する。

---

## 全体の流れ

```
① プロフィール設定（初回のみ・30分）
② 案件検索・選定（5〜10分/回）
③ 提案文作成・送信（15〜20分/件）
④ logs/projects.md に記録（2分）
⑤ フォローアップ（返信があった場合・5〜10分）
```

1件あたりの合計所要時間：**20〜30分**

---

## 1. プロフィール設定（初回のみ）

所要時間：**30分**

### 手順

1. CrowdWorks（https://crowdworks.jp）にログイン
2. マイページ → プロフィール編集 を開く
3. 各欄に以下を設定する

| 欄 | 設定内容 |
|---|---|
| タイトル | `profile/crowdworks-profile.md` の「タイトル」欄の文言をそのままコピー |
| 自己紹介文 | `profile/crowdworks-profile.md` の「自己紹介文」欄（```で囲まれた部分）をコピー |
| スキルタグ | `profile/crowdworks-profile.md` の「スキルタグ」欄に記載のタグを入力 |
| ポートフォリオURL | `https://github.com/Hiro4546M/freelance-portfolio` |

4. 保存して公開状態になっていることを確認する

> ポートフォリオが3本揃ったら、CrowdWorksの「ポートフォリオ」欄にGitHubリンクを追加する。

## 2. 案件検索・選定

所要時間：**5〜10分/回**

### 検索キーワード（優先順）

以下の順番でCrowdWorks検索窓に入力して案件を探す。
上位ほど高単価・競合少の傾向があるため優先する。

| 優先度 | キーワード | 理由 |
|---|---|---|
| 1 | `Make` | 競合が少なく高単価が多い |
| 2 | `n8n` | 同上。技術者が少ない |
| 3 | `Zapier` | 案件数が多い・やや競合あり |
| 4 | `業務自動化` | 幅広いが単価が分散する |
| 5 | `API連携` | 要件が具体的な案件が多い |
| 6 | `Slack通知` | 小規模だが即決しやすい |
| 7 | `Notion連携` | 案件数は少ないが需要は安定 |

### 受ける条件

以下をすべて満たす案件を選ぶ。

- 単価3万円以上（最初の2件は1万円以上でも可）
- 「〇〇を自動化したい」が明確な要件
- n8n / Make / Zapier / Slack / Notion / LINE / API連携 のどれかが含まれる
- クライアントが中小企業または個人事業主

### 受けない条件

以下に1つでも当てはまる案件はスキップする。

- ライティング・文章作成のみの案件
- 要件が「なんとなく自動化したい」レベルで具体性がない
- 単価1万円未満（実績3件達成後は絶対に受けない）
- サーバー構築・インフラ系（VPC設定・Linuxセットアップ等）
- 応募者数が20件を超えている（競争が激しすぎる）

## 3. 提案文の作り方

所要時間：**15〜20分/件**

### 使うテンプレート

`profile/proposal-template.md` の **テンプレートA（デモあり版）** を使う。
具体的な提案文サンプルは `profile/proposals/` 以下を参照。

| ファイル | 用途 |
|---|---|
| `profile/proposals/make-form-proposal.md` | Make フォーム→Slack/スプレッドシート案件 |
| `profile/proposals/n8n-automation-proposal.md` | n8n メール・AI連携案件 |
| `profile/proposals/api-integration-proposal.md` | 汎用API連携・要件不明確な案件 |

### 提案文の作成手順

1. 案件の要件文を読んで、解決したい課題を1行で言い換える（冒頭に使う）
2. 最も近い提案文サンプル（`profile/proposals/` 内）をコピーする
3. 以下の箇所を案件に合わせて書き換える

| 書き換え箇所 | 書き換え内容 |
|---|---|
| 冒頭の課題言い換え | 案件の要件を読んで自分の言葉で1行に要約 |
| 使用ツール | 案件に合ったもの（Make / n8n / Zapier / API等）を選ぶ |
| GitHubリンク | `https://github.com/Hiro4546M/freelance-portfolio` を記載 |
| 見積もり金額 | 案件の規模感に応じて設定（後述の目安を参照） |
| 確認事項 | 案件の不明点を1〜2点に絞って質問する |

4. 送信前に以下を確認する
   - [ ] クライアント名・案件タイトルに合致しているか
   - [ ] 金額・納期に矛盾がないか
   - [ ] GitHubリンクが正しいか
   - [ ] 誤字脱字がないか（読み返し1回）

### 見積もり金額の目安

| 規模 | 内容 | 金額 |
|---|---|---|
| 小 | 2〜3ステップの単純自動化 | 3〜5万円 |
| 中 | 複数ツール連携・条件分岐あり | 5〜15万円 |
| 大 | AI連携・複雑なロジック・設計込み | 15〜30万円 |

実績2件以内のうちは「小」案件で実績・評価を積むことを優先する。

### 送信のペース

- 週3件（月・水・金）に提案を送る
- 1日に複数件送ることは避ける（返信管理が煩雑になるため）
- 応募締切が近い案件は優先して対応する

## 4. logs/projects.md への記録

所要時間：**2分**

提案を送信したら、その日のうちに `logs/projects.md` に記録する。

### 記録手順

1. `logs/projects.md` を開く
2. 「進行中の案件」セクションに以下のフォーマットで追記する

```markdown
## [案件ID] 案件タイトル（CrowdWorksから引用）

- **プラットフォーム**：CrowdWorks
- **カテゴリ**：Make / n8n / API連携 など
- **提案日**：YYYY-MM-DD
- **単価**：¥XX,000
- **ステータス**：提案中
- **受注日**：（受注後に記入）
- **納品日**：（納品後に記入）
- **評価**：（納品後に記入）
- **作業時間**：合計 時間
- **デモ作成**：あり / なし
- **サンプル保存**：portfolio/samples/XXX（納品後に記入）
- **メモ**：（特記事項があれば）
```

3. 案件IDは `P001`、`P002`... と連番で付ける
4. ステータスが変わるたびに更新する（提案中→受注→納品済 / 不採用）

## 5. 提案後のフォローアップ

所要時間：**5〜10分/返信**

### 返信があった場合

**メッセージが来た当日中**に返信する（遅くとも24時間以内）。

#### パターン別の対応

| 状況 | 対応 |
|---|---|
| 詳細ヒアリングのメッセージ | 丁寧に回答し、不明点をさらに1〜2点確認する |
| 「見積もりをもう少し下げられますか」 | 10〜15%の値引きまでは応じる。それ以上は断る |
| 仕様変更の相談 | 変更内容によって追加費用が発生するか確認し、事前に合意を取る |
| 「他の方に決まりました」 | お礼を伝えて終了。logs に「不採用」と記録 |

#### ヒアリング返信のポイント

- 長文にしない（3〜5行以内）
- 質問は1〜2点に絞る
- 「いつ頃着手したいですか」を必ず確認する（スケジュール調整のため）

### 返信がない場合

- 提案から7日後に1回だけ軽くフォローする（任意）
- フォロー文例：「先日ご提案した件はご検討いただけましたでしょうか。ご不明点があればお気軽にご連絡ください」
- 14日以上返信がない場合は `logs/projects.md` のステータスを「不採用」とみなして更新する
